# ZenClean 网络连接问题排查指南

## 问题现象
点击"立即激活"时提示"网络连接失败，请检查网络"，但浏览器可以正常访问后台管理系统。

## 原因分析
这通常是Windows防火墙或杀毒软件拦截了ZenClean程序的网络请求导致的。

## 解决方案

### 方案一：运行网络诊断工具（推荐）

#### 方法A：在软件内进行诊断
1. 在授权页面输入激活码后点击"立即激活"
2. 如果连接失败，点击出现的"网络连接有问题？点击进行诊断"按钮
3. 等待诊断完成，查看详细的诊断报告
4. 点击"复制到剪贴板"将报告发送给技术支持

#### 方法B：运行独立诊断脚本
1. 双击运行 `诊断网络.bat`
2. 等待诊断完成，查看结果
3. 根据提示信息进行相应处理

#### 方法C：命令行运行诊断
```bash
python run_network_diag.py
```

### 方案二：手动检查防火墙设置

#### Windows 防火墙设置
1. 打开"控制面板" → "系统和安全" → "Windows Defender 防火墙"
2. 点击"允许应用或功能通过Windows Defender防火墙"
3. 点击"更改设置"（需要管理员权限）
4. 查找"ZenClean"或"Python"，确保勾选"专用"和"公用"网络
5. 如果没有找到，点击"允许其他应用"，添加ZenClean的exe文件

#### 常见杀毒软件设置
- **360安全卫士**：关闭"网络安全防护"或添加信任
- **火绒**：关闭"网络防护"或在"防护中心"中添加例外
- **腾讯电脑管家**：关闭"实时防护"中的网络防护

### 方案三：使用离线授权（紧急方案）

如果网络问题无法立即解决，可以使用离线授权模式：

1. 在浏览器中登录后台管理系统
2. 手动创建一个激活码
3. 在ZenClean中使用该激活码激活
4. 软件会自动缓存离线授权，24小时内无需联网

## 网络诊断工具说明

### 诊断内容
诊断工具会测试以下内容：

1. **DNS解析**：检查能否解析授权服务器的域名
2. **TCP连接**：检查能否建立到服务器的TCP连接
3. **SSL/TLS握手**：检查HTTPS加密连接是否正常
4. **HTTP请求**：检查能否发送HTTP请求

### 诊断结果解读

- **✅ 通过**：该项测试正常
- **❌ 失败**：该项测试失败，需要针对性解决

### 常见问题及解决

| 问题现象 | 可能原因 | 解决方案 |
|---------|---------|---------|
| DNS解析失败 | 网络连接问题或DNS配置错误 | 检查网络连接，更换DNS服务器 |
| TCP连接失败 | 防火墙拦截或服务器不可达 | 检查防火墙设置，添加例外规则 |
| SSL握手失败 | 系统时间错误或证书问题 | 校准系统时间，更新根证书 |
| HTTP请求失败 | 代理设置或网络策略限制 | 检查代理设置，联系网络管理员 |

## 验证修复效果

在进行上述修复后，重新运行诊断工具或尝试激活，确认问题是否解决。

## 获取技术支持

如果问题仍然无法解决，请：

1. 运行诊断工具并复制诊断报告
2. 截图防火墙/杀毒软件设置
3. 联系技术支持并提供上述信息

## 预防措施

1. 将ZenClean添加到防火墙白名单
2. 将ZenClean添加到杀毒软件信任列表
3. 定期检查网络连接状态
