# ZenClean 高级架构与性能参考手册 (v0.2.0)

本手册总结了 ZenClean 在处理系统 I/O、多任务并发及 Windows 深度集成方面的**进阶技术方案**。

---

## 🚀 1. 异步流式探测引擎 (I/O Streaming Pattern)

方案背景：全盘扫描或大规模 I/O 任务会导致 UI 卡死 1-2 秒。ZenClean 通过“边扫边出”消除了这种等待。

### 核心实现逻辑：
1.  **后端生成器 (Generator)**：在 `space_analyzer.py` 中使用 `yield` 逐步返回结果，而非集齐 100 个再返回。
2.  **异步桥接 (Async Bridge)**：
    *   使用 `asyncio.to_thread(next, gen)` 在后台线程中步进生成器，防止主线程（UI）被文件检索阻塞。
    *   通过 `await asyncio.sleep(0.01)` 给 UI 渲染留出“呼吸时间”。
3.  **三级跳加载 (Three-Stage Loading)**：
    *   **Level 1**: 离线快照 (JSON Cache) —— 0.1s 填充首屏。
    *   **Level 2**: 异步启动 —— 唤醒探针。
    *   **Level 3**: 流式排行榜重排 —— 每发现一个巨兽文件夹，实时重排 Top 5。

### 设计启示：
对于任何耗时超过 500ms 的数据获取任务，都应考虑“缓存先行 + 后台流式追加”的模式。

---

## 🛡️ 2. 事务级文件搬家 (Safe Migration)

方案背景：跨盘移动文件夹（如微信迁移）极易因中途断电或空间不足导致数据丢失。

### 安全策略：
1.  **Junction (符号链接) 技术**：将文件夹物理移动到 D 盘，在 C 盘创建虚拟节点。软件无需任何配置修改即可识别。
2.  **预检查机制**：
    *   **权限检测**：使用管理员提权令牌检测目标目录是否可写。
    *   **空间冗余度**：目标磁盘剩余空间必须 > 待搬移大小 + 2GB。
3.  **原子化移动尝试**：通过 `shutil.move` 进行移动，若失败则保留 C 盘原件并回滚临时目录。

---

## 🏗️ 3. 干净架构项目结构 (Clean Architecture)

即便是一个小型的桌面应用，ZenClean 也遵循了高度解耦的目录规范：

*   **`/src/ai`**：边缘智能模块，仅负责云端 API 交互，不包含任何业务判断。
*   **`/src/core`**：核心引擎层，纯 Python 实现，不依赖任何 UI 框架。
*   **`/src/ui`**：视图层，仅负责渲染。通过 `app` 对象调用 `core` 中的方法。
*   **`/src/config`**：全局动态配置中心，统一处理环境变量和路径。

### 价值：
这种结构使得我们可以轻松地将整个 `core` 模块迁移到 CLI 工具或 Web 服务的后端，而不必改动一行代码。

---

## 🖥️ 4. Windows 深度集成技巧

*   **硬件指纹**：`machineid.id()`。
*   **真实下载目录获取**：通过读取注册表 `HKEY_CURRENT_USER\...\User Shell Folders` 获取，而非硬编码 `C:\Users\...\Downloads`，适配了修改过默认下载位置的用户。
*   **非阻塞磁盘统计**：利用 `shutil.disk_usage` 快速获取分区信息。
