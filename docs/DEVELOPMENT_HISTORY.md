# ZenClean (禅清) 项目开发回顾

回顾过去的开发轨迹，我们将这个 Windows C 盘深度清理与优化工具从最初的一个概念，极速推进到了一个具备商业级软件雏形、且融合了 Flet 现代 UI 技术与本地/云端混合鉴定的工程。

开发过程主要分为以下几个清晰的篇章：

## 🏁 项目发端与基底设定
起初，项目灵感来源于传统的一键 C 盘清理脚本。为了彻底解决小白用户不敢删、高级用户嫌太慢的痛点，我们确立了全新的技术栈与产品定位：
- **技术选型**：Python 3.10 为引擎驱动，配合基于 Flutter 的 Flet 框架打造极具设计感的深色桌面端 UI。
- **核心策略**：引入混合 AI 鉴定（本地正则表达式知识库 + 云端大模型兜底），以及在文件底层访问规避系统崩溃的安全红线机制。

## 🚀 第一阶段：MVP版狂飙突进（已超前完成）
这一阶段原本定位为只做出基础交互，但实际上我们在极短的高压研发中把很多二期规划都提前拉满了：
1. **高性能并发扫描引擎**：在 `scanner.py` 中引入了 `multiprocessing` 独立子进程与 IPC 管道队列分发。解决了致命的 Windows `Junction`/`Symlink`（软链接/重解析点）死锁递归问题，实现了几十万文件 10 秒级扫毕，且主 UI（万级列表虚拟化渲染）帧率稳在 30fps。
2. **多层级“柔性”清理策略**：摒弃传统清理软件粗暴的删除法设定：
   - `LOW` (极低风险)：物理级抹除。
   - `MEDIUM` (中风险)：移入 Windows 回收站，给用户留后悔药。
   - `HIGH` (高风险)：锁定等待用户确认。
   - `CRISIS` (白名单/核心系统文件)：代码级熔断硬拒绝。
3. **商业化闭环与鉴权**：打通了自建的 `hw-license-center` 授权中心。通过提取用户底层硬件机器码，结合 JWT Token 缓存 + NTP 离线时钟防回退技术，实现了一套极难被破解的卡密激活链路。
4. **大文件夹无损搬移**：深度解析系统注册表，实现了用户层面呼声最高的“将整个桌面/下载/文档强行无损剥离并平滑移出 C 盘”。
5. **法务免责防线**：在代码层面接入了 `EULA` 免责拦截与 `PRIVACY_POLICY` 用户协议，为可能的误删提供前置法律屏障。

## 🛡️ 第二阶段：筑建商业护城河（当前正重点攻克）
目前整个工程已经步过功能探索期，进入到“商业分发前的极限防御修复期”。近期的记录显示，我们正在围绕以下硬核领域死磕：
1. **Windows 系统级特权与安全兜底**：
   - 筹划接入底层 Win32 API 自动申请 **UAC 管理员提权**。
   - 将在进行 `HIGH` 高危项清理前，前置启动调用系统级接口自动为用户 **创建 Windows 系统还原点**（SRSetRestorePoint），做到真·后悔药。
2. **云端智库与动态热修复**：
   - 近期我们就 AI 大模型调用的云端网关接口规划进行了深度讨论（例如确立 50 次/天的默认 AI 诊断额度配额等）。
   - 正在将死板的清理规则变成云端支持下发更新的 `file_kb.json` 热更中心。
3. **工程化打包分发 (PyInstaller)**：
   - 最新也是最棘手的一步：我们正在调试通过 `PyInstaller` 将 Flet 环境、多线程逻辑、资源一波带走。并且通过近期切转到 `--onedir` 目录打包模式，一并攻克 Windows 下的执行环境依赖与进程复制炸服问题。

---

**总结**
相比于很多简单的套壳工具，ZenClean 这个项目的开发过程表现出了极客般的技术克制（独立剥离极其严苛的守护白名单），又拥有极其现代的 GUI 思维（跨终端的极简卡片流交互），目前我们距离能够向公网独立分发一个完整、免安装且包含自我防卫能力的商业级 .EXE 执行文件仅有最后几步之遥。
