# ZenClean 已知问题与残留 BUG 追踪 (Known Issues)

本文件记录了 ZenClean 项目在当前版本中存在的细节瑕疵、潜在运行风险及设计上的权衡，作为未来优化的重点参考。

---

## 已修复 (Fixed)

### F-001: 授权返回值不一致导致崩溃 (V0.1.7-beta)
- **问题**: `verify_license_online` 部分错误路径返回 2-tuple，调用方解包 3 值时 ValueError。
- **修复**: 所有返回路径统一为 3-tuple，循环前初始化 `notification = None`。
- **文件**: `src/core/auth.py`

### F-002: 应用迁移页面调用不存在方法 (V0.1.7-beta)
- **问题**: `app_migration_view.py` 调用 `self.app.show_snack_bar()`，但 `ZenCleanApp` 无此方法。
- **修复**: `ZenCleanApp` 新增 `show_snack_bar(message, is_error)` 方法。
- **文件**: `src/ui/app.py`, `src/ui/views/app_migration_view.py`

### F-003: AI 缓存键不匹配 (V0.1.7-beta)
- **问题**: `_build_fallback_results` 按文件路径存储，`query()` 按父目录查询，缓存永远不命中。
- **修复**: 缓存存储统一改为按父目录键，与查询键一致。
- **文件**: `src/ai/cloud_engine.py`

### F-004: 缓存写盘静默跳过丢数据 (V0.1.7-beta)
- **问题**: 写盘锁被持有时直接 return，最近的 AI 判决可能丢失。
- **修复**: 引入 `_cache_dirty` 标记，写盘完成后检查并补写。
- **文件**: `src/ai/cloud_engine.py`

### F-005: .env 打包暴露基础设施 (V0.1.7-beta)
- **问题**: `zenclean.spec` 将 `.env` 打包进可执行文件，服务器地址可被提取。
- **修复**: 移除打包，改为运行时从 exe 同级目录或系统环境变量读取。
- **文件**: `zenclean.spec`, `src/config/settings.py`

## 待解决 (Open)

### 1. 系统搬家的非原子性风险 (设计局限)
- **问题**: `migration.py` 执行过程中若发生意外断电或程序崩溃，由于文件移动与注册表更新不是一个原子操作，可能导致"文件已走、路径未改"。
- **影响**: 导致 Windows 找不到用户文件夹。
- **当前状态**: 已实现断点恢复（5 阶状态机），但极端情况下仍有风险。

### 2. 统计数据与感知误差 (UI 体验)
- **问题**:
    1. **滞后感**: IPC 批次推送（`SCAN_BATCH_SIZE`）导致小文件密集区计数器跳跃。
    2. **容量对等性**: 清理后释放的空间统计可能与 Windows 磁盘属性的"可用空间"在 Byte 级别上不完全一致。
- **状态**: 低优先级。属于磁盘簇计算差异，可通过 UI 模糊化策略优化。

### 3. 云端鉴权的在线探活滞后
- **问题**: 后台解绑激活码后，客户端最长需要 30 分钟轮询才会强制退回到标准版。
- **风险**: 存在极短的"离线白嫖"时间窗。
- **拟定方案**: 在点击"开始扫描"的重操作点增加强制实时核验（已部分实现）。

### 4. JWT 签名验证未启用
- **问题**: 客户端 `jwt.decode` 显式禁用签名验证 (`verify_signature=False`)。
- **原因**: 服务端 `ZC_API_SECRET` 尚未同步部署。
- **风险**: 任何客户端可离线构造有效 JWT。
- **拟定方案**: 服务端部署 Secret 后启用强验签。

### 5. Scanner 停止标志粒度不足
- **问题**: 停止标志仅在目录级检查，大目录（百万文件）内响应停止请求较慢。
- **拟定方案**: 增加文件级定期检查（如每 100 个文件）。
