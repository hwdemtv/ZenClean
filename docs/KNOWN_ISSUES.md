# ZenClean 已知问题与残留 BUG 追踪 (Known Issues)

本文件记录了 ZenClean 项目在当前版本中存在的细节瑕疵、潜在运行风险及设计上的权衡，作为未来优化的重点参考。

---

## 1. 系统权限与清理深度 (待解决)

*   **问题描述**：程序启动后若未以“管理员身份”运行，扫描引擎会因 `PermissionError` 跳过大量系统目录（如 `C:\Windows\Temp`）。
*   **用户影响**：小白用户可能感官上觉得“清理不彻底”，导致产品口碑受损。
*   **拟定方案**：优先级 1 任务——实现 UAC 自动提权逻辑。

## 2. 打包环境下的子进程崩溃风险 (工程细节)

*   **问题描述**：在 Windows 上使用 `PyInstaller` 打包后，如果 `multiprocessing.freeze_support()` 配置不当，启动扫描会引发主进程不断重启的“递归炸弹”。
*   **状态**：待最终打包压测。需确保 `main.py` 入口点极致健壮。

## 3. 系统搬家的非原子性风险 (设计局限)

*   **问题描述**：`migration.py` 执行过程中若发生意外断电或程序崩溃，由于文件移动与注册表更新不是一个原子操作，可能导致“文件已走、路径未改”。
*   **影响**：导致 Windows 找不到用户文件夹。
*   **拟定方案**：增加搬家中间状态日志，实现下次启动时的“自动修复/续传”功能。

## 4. 统计数据与感知误差 (UI 体验)

*   **问题描述**：
    1.  **滞后感**：IPC 批次推送（`SCAN_BATCH_SIZE`）导致小文件密集区计数器跳跃。
    2.  **容量对等性**：清理后释放的空间统计可能与 Windows 磁盘属性的“可用空间”在 Byte 级别上不完全一致。
*   **状态**：低优先级。这属于磁盘簇（Cluster）计算差异，可通过 UI 模糊化策略（如显示“约 XX GB”）来优化体感。

## 5. 云端鉴权的在线探活滞后 (鉴权细节)

*   **问题描述**：目前后台解绑激活码后，客户端最长需要 30 分钟轮询才会强制退回到标准版。
*   **风险**：存在极短的“离线白嫖”时间窗。
*   **拟定方案**：在点击“开始扫描”的重操作点增加强制实时核验（已部分实现）。
