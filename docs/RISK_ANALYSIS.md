# ZenClean 项目风险分析报告

作为一款涉及 Windows 系统级文件操作的清理软件，安全性是 ZenClean 的生命线。本报告对当前项目的风险点、现有防御机制及改进建议进行了全面梳理。

## 1. 核心风险点分析

### A. 误删系统关键文件 (False Positives)
*   **风险描述**：清理规则或 AI 判定失误，导致 Windows 核心 DLL、系统驱动或关键配置文件被删，引发蓝屏或系统无法启动。
*   **当前表现**：风险极低。目前已接入 `whitelist.py` 绝对白名单守卫。

### B. 系统搬家后的路径丢失 (Registry Desync)
*   **风险描述**：在“系统搬家”执行过程中，若文件搬迁完毕但注册表更新前程序崩溃，会导致 Windows 找不着桌面/文档路径。
*   **当前表现**：中。目前使用了 Junction Link (J) 软链接补偿方案和 Registry Rollback 备份机制。

### C. 权限升级风险 (UAC Risk)
*   **风险描述**：扫查或清理过程中因权限不足（未提权）导致操作失败，或者提权后程序被恶意脚本利用执行非法操作。
*   **当前表现**：中。目前依赖用户手动管理员运行，尚未自动化提权。

### D. 隐私与数据安全
*   **风险描述**：提取 MachineID 进行鉴权可能引起敏感用户的担忧。
*   **当前表现**：低。MachineID 仅用于本地/服务端强绑定，不涉及个人身份数据。

---

## 2. 现有防御机制 (Security Layers)

| 防御层级 | 实现位置 | 功能说明 |
| :--- | :--- | :--- |
| **绝对白名单** | `whitelist.py` | 锁定 `System32`、`WinSxS` 等 10+ 个核心目录，屏蔽 `.sys`/`.dll` 等关键后缀。 |
| **分级清理策略** | `cleaner.py` | `LOW` 直接删，`MEDIUM/HIGH` 进回收站（后悔药），`CRISIS` 强制拦截。 |
| **迁移前置核验** | `migration.py` | 确保目标盘空间足够（10% 余量），且防止父子目录循环移动。 |
| **原子性搬家** | `migration.py` | 采用“先搬文件、后改注册表”的顺序，确保路径切换时数据已就绪。 |

---

## 3. 残余风险与改进建议 (Residual Risks & TODO)

### 🚨 重点改进项：管理员权限自动化 (UAC)
*   **风险**：目前扫描不全、清理报错多由于权限不足。
*   **建议**：在 `main.py` 增加自检提权逻辑，确保引擎运行在 Full Token 下。

### 🚨 终极防御：系统还原点 (System Restore)
*   **风险**：即便有白名单，依然可能存在极端情况下的误配。
*   **建议**：在 VIP 执行“一键清理”前，利用 WebBrowser 或 Win32 调用自动创建一个系统还原点。

### 🚨 搬家容错性增强
*   **风险**：`rollback()` 目前仅回滚注册表，不移动文件。
*   **建议**：完善完整的回滚链路，支持双向路径镜像还原。

---

**结论：**
ZenClean 的架构在**设计初衷**上是非常保守且安全的。目前已有的四层防御足以应对 95% 以上 of 误操作风险。后续工作的重点应放在**“提升自动化操作权 (UAC)”**与**“增强容灾手段 (还原点)”**上，以对冲最后 5% 的极端风险。
