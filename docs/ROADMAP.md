# ZenClean 项目后续开发路线图 (Roadmap)

根据“开发投入”与“产出价值”（性价比）的综合评估，我们对 ZenClean 剩余的核心任务进行了优先级排序。这套路线图旨在通过最小的边际成本实现最大的产品力提升。

---

## 🥇 优先级 1：UAC 管理员提权自触发 (最高的 ROI)

*   **投入**：**极低**
    *   仅需在 `main.py` 启动逻辑中接入 `ctypes` 调用检测，或在打包时配置 Manifest 清单。
*   **收益**：**极大**
    *   **核心痛点解决**：目前由于没有提权，许多 C 盘系统目录扫描不到、锁定文件删不掉。提权将直接决定工具的“底层清洁力”，是所有功能的基础。
*   **结论**：这是目前最应优先完成的“一劳永逸”型基础设施。

## 🥈 优先级 2：规则库云端热更新 (产品生命力)

*   **投入**：**中等**
    *   前端增加简单的 HTTP 下载逻辑；后端维护一个静态 JSON 规则文件。
*   **收益**：**极大**
    *   **响应速度**：让软件具备“生命力”。无需频繁发布新版本，只需更新云端 JSON，即可识别最新出现的流氓软件或临时缓存路径。
*   **结论**：这是支撑产品商业化长期运作、保持竞争力的核心。

## 🥉 优先级 3：深度清理视图 (极致下潜能力)

*   **投入**：**较高**
    *   新增独立扫描引擎策略，开发专属 UI 页面，处理更为复杂的依赖图谱和安全警告。
*   **收益**：**极大**
    *   **满足极客诉求**：为核心玩家解锁彻底清理解构级缓存（如 `.NET Framework`、`Installer` 等）的能力，作为常规“智能清理”的火力补充。
*   **结论**：这是增强产品硬核技术标签、拉开与竞品差距的高级功能。

## 🏅 优先级 4：打包分发优化 (功能的放大器)

*   **投入**：**较高**
    *   Windows 环境下的路径寻址、子进程 (multiprocessing) 打包非常复杂，需要长时间的兼容性调试。
*   **收益**：**极大**
    *   **产品化转码**：这是软件从“实验代码”变为“商业产品”的必经之路。解决环境依赖，让用户开箱即用。
*   **结论**：在正式发布前必须解决，是交付能力的终极体现。

## 🎗️ 优先级 5：系统还原点自动创建与“时光机”恢复舱 (安全生命线)

*   **投入**：**中等**
*   **收益**：**极大**
*   **核心痛点解决**：单一的回收站机制对小白用户不够直观且难以恢复被锁定的深层文件。
*   **实现路径**：
    1. 底层调用 `ctypes.windll.srclient.SRSetRestorePointW` 创建 Windows 系统快照。
    2. 应用层基于 `%APPDATA%\ZenClean\Quarantine` 维护离线沙箱。
    3. UI 层开发专属的 **「TimeMachine (时光机)」视图**，以时间轴形式展现清理记录，提供醒目的“一键反悔”按钮。
*   **结论**：这是产品进入小白市场前的法律与品牌安全保障，也是体现专业度的标配功能。

## 🌟 优先级 6：Dashboard 数据大屏与用户体验革新 (视觉张力)

*   **投入**：**较高**
*   **收益**：**极大 (转化率与留存)**
*   **核心目标**：将原先极简的列表展示，升级为极具科技感与视觉表现力的数据面板。
    1. **Dashboard (大盘视图)**：利用 Flet Chart (`ft.PieChart`, `ft.BarChart`) 生成全盘/C盘的存储状况，精准区分 视频/图片/代码包/缓存。
    2. **安全风险视觉强化**：针对不同风险标记 (安全/警告/危险) 的文件，采用极具冲击力的红黄绿配色进行物理隔离展示。
    3. **个性化设定 (Settings)**：提供完整的暗夜/明亮模式切换、以及紧凑列表与宽屏卡片等多种 UI 布局控制参数。
