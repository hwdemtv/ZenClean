# ZenClean 项目后续开发路线图 (Roadmap)

根据“开发投入”与“产出价值”（性价比）的综合评估，我们对 ZenClean 剩余的核心任务进行了优先级排序。这套路线图旨在通过最小的边际成本实现最大的产品力提升。

---

## 🥇 优先级 1：UAC 管理员提权自触发 (最高的 ROI)

*   **投入**：**极低**
    *   仅需在 `main.py` 启动逻辑中接入 `ctypes` 调用检测。
*   **收益**：**极大**
    *   **核心痛点解决**：没有提权，许多 C 盘核心目录就无法扫描。提权决定了清理的“底层杀伤力”。

## 🥈 优先级 2：Dashboard 可视化大盘与系统极速瘦身 (最高的用户感知 ROI)

*   **投入**：**中等**
    *   UI 层引入 Flet Chart 绘制全盘存储结构图；系统层只需调用 `powercfg -h off` 和展示虚拟内存。
*   **收益**：**爆表感官**
    *   一键关闭休眠直接腾出 8~16 GB 空间，是清理垃圾难以企及的“物理快感”。炫酷的大屏图表直接能拉开软件档次。
*   **结论**：这是让产品在小白眼中显得极具商业竞争力的“面子工程”，应列为阶段二的核心宣发卖点。

## 🥉 优先级 3：专属「TimeMachine (时光机)」容灾沙箱 (安全底座)

*   **投入**：**较高**
    *   底层的 `SRSetRestorePointW` 容错；应用层的 `%APPDATA%\ZenClean\Quarantine` 日志维护和文件迁移逻辑，以及全新的时间轴视图。
*   **收益**：**极大 (防御性)**
    *   解决极端环境下的灾难性客诉。
*   **结论**：开发难度大但必须做，作为阶段三（护城河）的重头戏。

## 🏅 优先级 4：第三方应用搬家与僵尸环境猎手 (深度拓展)

*   **投入**：**较高**
    *   处理 Junction 两端映射，扫描深海级 `node_modules` 的耗时分析。
*   **收益**：**中等** 
    *   针对小部分重度开发/软件囤积用户的长尾需求。

## 🎗️ 优先级 5：规则库云端热更新与自升级 (生命线)

*   **投入**：**中等**
    *   构建无感知的云端 JSON 校验、替换机制。
*   **收益**：**长远巨大**

---

## 🛠️ 后续执行计划蓝图 (基于最高性价比)

1.  **[阶段二: 视觉与特权突破]** 
    *   **目标**：用最小的代码成本，换取最震撼的空间释放（休眠文件）和最极客的图表（Dashboard），同时拿下系统最高操作权（UAC）。
    *   **特性**：UAC 唤醒、系统极速瘦身（一键+16G）、空间仪表盘大屏、个性化设置 (暗黑模式)。
2.  **[阶段三: 铁壁沙箱舱]** 
    *   **目标**：建立绝对的安全反悔机制，并处理沉淀业务数据。
    *   **特性**：TimeMachine 恢复舱维护、应用软连接搬家。
3.  **[阶段四: 商用化发版]** 
    *   **特性**：热更引擎 (`OTA`)、商业数字签名防毒免疫、自动封包安装程序。
