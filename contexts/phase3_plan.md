# 阶段三 (完全体与自动值守版) 实施方案 v1.0

回顾前两个阶段，ZenClean 已经具备了极速扫描、本地规则匹配、真实云端大模型 SSE 流式网关对接、完善的设备指纹授权、双重清理（回收站保护）、注册表无损搬家等强大核心。开发进度远超预期，底层基建已牢不可破。

进入**第三阶段（终局冲刺）**，我们的核心目标是从“好用的极客工具”向“**极其惊艳的商业级 SaaS 桌面软件**”蜕变。这要求我们在**交互感知、自动化驻留、操作系统级集成**以及**打包免杀分发**这四方面进行最后合围。

---

## 1. 核心任务定义 (Task Clusters)

### 战役一：极致体验与动效反馈 (UX/UI Polish)
清理软件的核心卖点是“释放空间的爽感”，必须将这种视觉冲击力做到极致。

*   **Task 1.1: 智能一键体检流程**
    *   **目标**：彻底改造目前的极客风格勾选逻辑。在首页引入一个巨大的“闪电体检”按钮。
    *   **实现**：一键聚合 `[启动扫描 ➜ AI规则裁决 ➜ 云端大模型判定 ➜ 自动预勾选LOW/MEDIUM风险文件]` 流程。抛开繁琐分类，直接给用户一个“极简的体检分数与总释放空间预测”。
*   **Task 1.2: 空间释放水柱动画**
    *   **目标**：当执行物理清理或移入回收站后，顶部仪表盘必须提供极具张力的动态反馈。
    *   **实现**：利用 Flet 动画组件，使环形进度条在清理瞬间发生带弹簧效应 (Spring) 的容量回退，同时触发“变绿”视觉奖励机制。
*   **Task 1.3: 开机爆仓气泡预警 (System Tray)**
    *   **目标**：在后台轻量级轮询系统盘剩余空间。
    *   **实现**：当 C 盘空间低于 10% 或 10GB 时，在右下角系统托盘弹出自定义炫酷气泡：“C 盘空间已亮红灯，点击一键唤醒 ZenClean”。

### 战役二：高级商业化独占特性 (VIP Exclusives)
为了刺激用户从免费白嫖或每日打卡向年费/买断付费转化，必须提供省时省力的特权。

*   **Task 2.1: 定时静默爆仓清理任务 (Windows Task Scheduler)**
    *   **目标**：VIP 用户可开启“全自动值守”。
    *   **实现**：通过 Python 调用 `schtasks` 或直接注册 Windows 计划任务，让 ZenClean 在每周闲时或 C 盘报警时，不弹 GUI 界面直接以后台进程 (`main.py --silent`) 执行安全层级下的自动清理。
*   **Task 2.2: 终局版真实卡密校验接入**
    *   **目标**：告别每日公码，对接真正的独立发卡逻辑。
    *   **实现**：配合 `hw-license-center` 端点，基于动态生成并与用户名绑定的独占激活码，实现一机一码永不掉签，彻底剥离微信找码流程。

### 战役三：操作系统深度集成 (OS Integration)
让用户在处理日常文件时能随时体验 AI 引擎的威力。

*   **Task 3.1: 右键菜单集成 (Context Menu)**
    *   **目标**：给任意文件夹右键增加“ZenClean 智能粉碎分析”选项。
    *   **实现**：通过修改注册表 `[HKEY_CLASSES_ROOT\Directory\shell]`, 将软件路径与 `%1` 参入绑定，实现唤起 Flet 应用并直接锁定分析目标。
*   **Task 3.2: 拖拽区域支持 (Drag & Drop)**
    *   **目标**：支持跨平台桌面端极客玩法。
    *   **实现**：在应用主界面实现一个拖拽释放区 (`file_picker` 或 Flet `on_drop`)，任何拖入的未知 `.cache` 文件夹瞬间对接本地文件知识库或发送给 GLM-4-Flash 进行无情裁定。

### 战役四：封顶打包发布与合规防杀 (Distribution)
完成最后一公里的商业化护驾包。

*   **Task 4.1: 日志全自动清理策略**
    *   **目标**：清理软件自身不能成为勒索空间的垃圾。
    *   **实现**：强化 `logger.py` 中 7 天到期淘汰机制，加入 `loguru` 式的文件截断滚动，或者依托启动时的自我打扫模块清理 `%AppData%\ZenClean\logs\`。
*   **Task 4.2: Inno Setup 优雅安装向导编译**
    *   **目标**：告别单体 `.exe` 的解压卡顿与无图标现象。
    *   **实现**：使用 `PyInstaller --onedir` 模式编译，配合 Inno Setup 生成 `ZenClean_Install_v1.0.exe`。包含同意协议 (`EULA`) 勾选、创建桌面快捷方式与 UAC 管理员提权清单锁定。
*   **Task 4.3: EV 证书签名与杀毒免疫 (计划指导)**
    *   对安装包与主程序执行代码签名；撰写一份申请 360/火绒白名单的材料说明模板，应对后续软件在 C 盘系统根目录下挥刀可能带来的误杀风控。

## 2. 实施路线图 (Timeline)

- **Week 1 (动效与集成)**: 搞定首页“一键体检”重构，完善 ListView 的动画释放反馈，注册并测试 Windows 右键菜单与拖拽功能。
- **Week 2 (自动化与提权)**: 开发 `--silent` 静默清理模式 CLI 入口，集成 Windows 计划任务。
- **Week 3 (打包完结)**: 切换 PyInstaller 生成策略为 `onedir`，编写 Inno Setup 脚本，测试安装/卸载逻辑（确保不残留配置文件）。完成 V1.0 RC 候选版封装。

---

## 3. 验收标准
1. **一键体检顺滑**：毫无计算机基础的小白用户，点击大按钮后能在 30 秒内顺滑看完扫描与 AI 裁决结果汇总，并安全释放空间。
2. **免打扰清理**：通过配置好计划任务，能在无人在机前的情况下，检测并默默清空 LOW 级系统垃圾并留痕于 logs 中。
3. **分发专业化**：产出一个带有安装全流程界面的标准 Windows `.exe` 安装包。
