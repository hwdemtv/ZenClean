# 阶段三 (护城河防御与商业分发) 实施方案 v1.0

回顾阶段一与阶段二，ZenClean 已经具备了极速扫描、UAC 底层提权、惊艳的数据大屏与一键系统瘦身能力。作为一款本地小工具，它的表现力已经达到极致。

进入**第三阶段**，我们的目标是从“好用且酷炫的极客工具”向“**极其安全可靠的商业级 SaaS 桌面软件**”蜕变。这要求我们在**灾备兜底、深度场景拓展**以及**正规化法务与分发**上构建坚固的壁垒。

---

## 1. 核心任务定义 (Task Clusters)

### 战役一：安全底座与“时光机”恢复舱 (TimeMachine)
对于一款敢动 C 盘底层文件的管家类软件，真正的考量不在于能扫出多少垃圾，而在于“误删后能否无伤救回”。

*   **Task 1.1: 创建系统还原点 (SRSetRestorePointW)**
    *   **目标**：在用户执行含 `HIGH` 高危文件的清理前，强行插入 Windows 系统级快照。
    *   **实现**：调用底层 API。需使用异步后拉线程与超时(Timeout)切断机制，防止 VSS 服务故障导致主程序卡死。
*   **Task 1.2: 离线物理沙箱备份 (Quarantine)**
    *   **目标**：接管原本丢入系统回收站的中低风险文件，转为内部隔离。
    *   **实现**：将文件移动至 `%APPDATA%\ZenClean\Quarantine`。
*   **Task 1.3: 专属时光机视图**
    *   **目标**：开发 `TimeMachineView` 面板。
    *   **交互**：以时间轴呈现历史清理批次，提供“一键反悔”按钮将沙箱文件原路复原。

### 战役二：空间深度拓展：三方应用搬家
系统瘦身虽猛，但极客和内容创作者的 C 盘往往是被微信、QQ、Docker 等常驻生态塞满的。

*   **Task 2.1: 通用应用搬家引擎 (`app_migrator.py`)**
    *   **目标**：将大型第三方应用数据无损迁移至 D 盘。
    *   **实现**：底层基于 Windows 目录交汇点 (Junction Point)。将源文件夹剪切后，在原位创建一个软指令向新地址。
    *   **UI 联动**：在空间管理聚合页新增 App Migration 面板，提供知名软件的热区扫描勾选。

### 战役三：规则的生命力：云端热更体系 (OTA)
AI 虽然聪明，但响应速度和成本永远比不上本地正则。防范新的流氓软件，依赖规则库的鲜活性。

*   **Task 3.1: `file_kb.json` 规则树热更**
    *   **目标**：启动时静默从 CDN/防篡改网关拉取最新 JSON 替换本地静态缓存。
    *   **防护**：必须包含 SHA256 签名硬校验，防止中间人下发投毒规则（如恶意认定 System32 为低级垃圾）。

### 战役四：封顶发版与合规防杀 (Distribution)
完成最后一公里的商业化护驾包。

*   **Task 4.1: Inno Setup 优雅安装向导**
    *   **目标**：告别单体 `.exe` 解压慢与不正式的缺点。
    *   **实现**：使用 `PyInstaller --onedir` 模式编译，配合 Inno Setup 生成 `ZenClean_Install_v1.0.exe`。包含同意协议 (`EULA`) 与管理员提权清单绑定。
*   **Task 4.2: EV 商用数字签名**
    *   **目标**：消除运行时“未知发布者”的安全黄盾警告。
    *   **执行**：采购 EV 代码签名证书，加入打发行包流水线集成。申请 360/火绒安全厂商白名单，根治系统优化软件必然面临的误杀风控。
