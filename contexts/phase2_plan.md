# 阶段二 (无界提权与极客大屏) 实施方案 v2.0 (ROI 优化版)

回顾第一阶段（MVP版），我们的开发进度实际上大幅超前。基础扫描与硬链接搬家框架均已稳固。

为了让软件**最快速地具备强烈的商业震撼力**，我们按照“开发成本最低、用户感知最强 (ROI)” 的原则，对真实的第二阶段目标进行了大幅优化重组。我们将那些“防微杜渐但吃力不讨好”的护城河机制（如系统快照与沙箱）延后至三期，**在阶段二将火力全部集中于：突破系统权限制约，引入发光数据大屏，以及实现秒增十 G 空间的物理释放。**

---

## 1. 核心任务定义 (Task Clusters)

第二阶段分为四大核心战役，旨在直击用户视觉与清理爽感的爽点：

### 战役一：核心特权突破 (UAC Escalation)
这是软件彻底告别“玩具”标签，能够清扫 C 盘深水区的绝对基石。

*   **Task 1.1: 智能 UAC 提权唤起**
    *   **目标**：在清理核心路径时获取 `Administrator` 权限。
    *   **实现**：在 `main.py` 入口点探测 `ctypes.windll.shell32.IsUserAnAdmin()`。若为假，则通过 `ShellExecuteW(..., "runas")` 动词发起 UAC 白盾弹窗要求重载进程。
    *   **难点排除**：需要针对 PyInstaller `multiprocessing` 模块在提权重启时防范子进程炸弹 (Fork Bomb)。

### 战役二：极具张力的空间大屏 (Dashboard 视图)
从简陋的列表流升级为具备发光特效与图形化感知的数据面板，是打动小白与极客共同的杀手锏。

*   **Task 2.1: 存储结构多维图表 (Flet Charts)**
    *   **目标**：在 `SpaceView` 引入 `ft.PieChart` 或 `ft.BarChart`。
    *   **实现**：扫描后，以图表形式精准展示“图片/视频”、“系统缓存”、“开发临时包”分别吞噬了多少 G 空间。利用红黄绿配色构建起强烈的风险隔离感。

### 战役三：API 全域消息与版本广播 (方案二)
在每次鉴权心跳或验证卡密时，顺带建立高触达率的运营/升级通知通道。

*   **Task 3.1: 解析云端 Notification 载荷**
    *   **目标**：无缝接收 `hw-license-center` 下发的系统公告、活动或强提醒版本更新。
    *   **实现**：修改 `core/auth.py`。当检测到后端返回 JSON 中含有 `notification` 块时，抛出全局事件。
    *   **UI 联动**：在首页顶端渲染可关闭的 Snackbar 或在系统右下角弹出 Toast，引导用户点击相应的 `action_url` (如跳转新版本下载页)。

### 战役四：系统的极速极致瘦身 (System Optimizer)
提供一种抛开常规“垃圾删除”，能够闭眼瞬间压榨出极大空间的快捷能力。

*   **Task 4.1: 休眠文件的生杀大权**
    *   **目标**：一键切换 `hiberfil.sys` 的存废，瞬间释放 8~32 GB 大小。
    *   **实现**：调用底层命令 `powercfg -h off/on`。并在 UI 上结合仪表盘卡片做显眼操作。
*   **Task 4.2: 虚拟内存迁移引导**
    *   **目标**：探测 `pagefile.sys` 体积并指明系统原生迁移入口。
    *   **实现**：探测数值后，提供按钮一键打开系统 `sysdm.cpl` (高级系统设置) 引导用户手动挪走。

### 战役五：质感跃升与极客定制 (Settings)
给软件披上专业的“皮肤”。

*   **Task 5.1: 全局暗黑模式与布局控制**
    *   **目标**：开发 `SettingsView` 页面。
    *   **实现**：加入亮/暗色主题一键无缝热切换 (`page.theme_mode`)；加入“列表密度调整”选项，允许用户在“紧凑模式(Compact)”（适合几万条结果的极客）和“舒适模式”间自由选择。

## 2. 实施路线图 (Timeline)

- **Week 1 (攻坚特权与瘦身)**：优先拿下 `main.py` 的 UAC 提权逻辑，并编写 `System Optimizer` 封装脚本以管理休眠文件。
- **Week 2 (面子工程)**：开发 Dashboard 大盘，利用 Flet Chart 将后台的扫描数据灌入图表中。开发 Settings 主题面板。
- **Week 3 (集成与打包)**：确认新版本在普通单用户下能正常拉起黄盾并弹窗，完成构建包含提权 Manifest 的 `ZenClean_v0.2.0` 测试包发布。

---

## 3. 为什么将“时光机快照”移出本期？
基于 ROI 分析：`SRSetRestorePointW` (系统快照) 开发调试极度困难（涉嫌服务阻塞、线程卡死），且对正常流程的用户“零感知”。它属于阶段末期的合规安全锁，而不该在抢产品化商业眼球的冲刺期（阶段二）占用宝贵带宽。因此我们将其移入阶段三继续打磨。
