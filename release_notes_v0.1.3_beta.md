# ZenClean v0.1.3 Beta - 引擎降维打击与视界解放

本次版本是自立项以来涉及最底层和最顶层的高维大满贯跨越！从隔离扫描引擎脱离 UI 进程的“大心脏”换手大手术，到防刷签名的护航，再到系统级常驻的右键提权与托盘；甚至彻底重组了渲染流水线，将所有残缺重叠的视觉瑕疵“挫骨扬灰”。
这不仅是 Beta版的收官，更是对即将到来 v1.0 商业化的全方位演习。

## ✨ 新特性 (Features)

*   **多核隔离引擎 (Scanner 2.0)**：彻底摒弃 Flet 挂载线程的旧有同步扫盘，将检索器剥离为完全独立的后台隔离线程+异步通讯钩子。几万级别的文件盘点自此与 GUI 的刷新解耦，彻底终结了“假死卡顿”时代。
*   **云端反撸护城河 (HMAC-SHA256)**：在 API 请求层实装了时间戳哈希校验+Nonce 随机数，给请求接口戴上了严丝合缝的安全套接字，使得模拟发包与脱机刷激活码无处遁形。
*   **深层系统锚点 (System Deep Anchor)**：打通并内置了右键菜单清理快捷挂载，系统开机驻留监控预警气泡，以及常驻后台的智能系统托盘（Tray），全面入侵 Windows 系统级高优权限体验区。
*   **极视界流式网格 (Fluid Grid Legend)**：针对大文件分类仪表盘独创性采用了 `Wrap` 动态包裹渲染算法，将 `PieChart` (环形甜甜圈图) 与所有图例文字分列。当扫描了十几个繁杂的分类时，图例会自动形成完美排列的多列矩阵，告别了极其难看的一维重叠崩溃。

## 🎨 界面与体验跃升 (UI/UX)

*   **巨幕弹性自留地 (Expand Cascade)**：修复了宽屏 / 高清屏由于父对象高度未释放导致的卡片不顶满、底部留白的问题。赋予了 `Column` 绝对的高度霸占参数 (`expand=True`)，让高分辨率屏幕下的清理列表如海绵般吃空多余画幅，绝不浪费一厘！
*   **悬浮页脚定海神针**：“一键清理大蓝按钮”被彻底脱离了普通的滚动流中，将其焊死作为所有卡片列表下方的雷打不动的“定海器”。即使用户在上方如何翻江倒海滚列表条目，清理按键都不可能再遮挡任何一行字迹。
*   **隐形底边防撞海绵**：此前最痛苦的 Flet 滚动列表中最后一张卡片永远被裁切、吃边等问题，由于底层改换原生的自动流引擎，得到了一刀切的斩首解决。

## 🛠 隐患剪除 (Fixes)

*   **ListView 幽灵截断 Bug 彻底破案**：弃用 `ListView` 的虚拟高度预估算法，因为在遇到 `ExpansionTile` 这种能一秒长出几百高度的可折叠卡卷时会导致卷轴长度计算死锁，让所有的深坑残缺死角迎刃而解！
*   修复了 Inno Setup 与 PyInstaller 打包锁死的跨时序残留报错。
*   大幅下架冗杂控制台警告包，缩紧第三方模块依赖库，缩小封包厚度。
